# SPEC — 季節のご挨拶メール（年2回）

## 成果物
- メール本文（件名＋本文）：クライアント宛のご挨拶メール文面
- Slack 用文面：一部クライアント向けに、同趣旨を Slack のトーンで調整したもの
- 宛名差し込み前のテンプレート＋差し込み後の個別文面

## 構成要素（冬・夏 共通）
1. 件名（例：【株式会社mijica】〜のご挨拶ならびに〜休業のご案内）
2. 宛名（会社名・担当者名）
3. 時候・感謝の挨拶（差出人：畑田）
4. 休業案内ブロック（罫線で囲む）
   - 最終営業日 / 営業開始日 / 休業期間
   - 緊急時は各営業担当まで、の一文
5. お歳暮／お中元の送付案内＋直接訪問できないお詫び
6. 結び（今後の変わらぬご愛顧のお願い、季節に応じた締め）

## 冬版と夏版の違い
| 項目 | 冬（年末年始） | 夏 |
|------|----------------|-----|
| 挨拶 | 年度末・年の瀬 | 盛夏・暑中 |
| 品物 | お歳暮 | お中元 |
| 休業 | 年末年始休業 | 夏期休業（お盆） |
| 結び | 良いお年を／ご自愛 | 猛暑の折ご自愛 |

## 媒体別の調整
- **メール**：前回踏襲のかしこまったトーン。罫線ブロックをそのまま使用
- **Slack**：やや柔らかめ。罫線は簡易な箇条書きに。宛名は Slack の呼びかけに合わせる

## 制約・注意
- 送信は必ず内容を確認してもらってから（勝手に送らない）
- 休業日程・宛名は毎回変わるため、送付前に最新の日程を確認する
- 送付先リストは個人情報のため input/ 管理、リポジトリにコミットしない配慮
