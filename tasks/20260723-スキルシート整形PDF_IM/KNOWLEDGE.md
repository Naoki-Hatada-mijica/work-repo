# KNOWLEDGE — 20260723-スキルシート整形PDF_IM

このタスクで skillsheet-excel-to-pdf スキルを3点アップデートした。以降のPDF化にも自動で効く。

## 1. Excelの色が全部白く飛んでいた（最大のハマりどころ）

Excelの「塗りつぶしの色」パレットから選んだ色は、色の数値（RGB）ではなく
**「テーマ色の番号＋濃さ（tint）」という間接的な形**で保存される。
openpyxl の `color.rgb` はテーマ色だと空を返すため、対応していないと**色が全部消える**。
今回の経歴書も93セルが薄い青だったのに、真っ白で出力されていた。

対応（`xlsx_to_html.py` の `argb()` / `apply_tint()`）:
- 各ファイルの `theme1.xml` の配色（clrScheme）を読む
- テーマ番号→配色名は **0/1・2/3 が入れ替わる**（0=lt1, 1=dk1, 2=lt2, 3=dk2, 4〜9=accent1〜6）
- 濃さ（tint）は明度を `tint<0: L*(1+tint) / tint>0: L*(1-tint)+tint` で寄せる
- 検証：Excel既知値（accent1 `4472C4` ＋tint0.6 = `B4C7E7`）と計算結果が一致
- インデックス色にも対応

## 2. 行間はユーザー好みで「ゆったり」

`ROW_SCALE` 0.9→**1.22**、`LINE_HEIGHT` 1.12→**1.6**、`CELL_PAD_Y` **2px**。
この3つはセットで動かす（片方だけだと文字が枠に接する）。文字サイズは変えない。
**ページ数が増えても読みやすさを優先する**のが好み（1.06/1.32では「まだ狭い」と再指摘された）。

## 3. 罫線の強弱（何往復かして固まった仕様）

点線にするのは **「案件ブロック」×「業務内容列から右」だけ**。

| 場所 | 線 |
|---|---|
| 外枠・案件の切り替わり | 実線（最低1.2pxに太らせる） |
| 項番(No)・期間の欄 | 実線（案件の中でも太線のまま） |
| 業務内容〜使用言語・サーバ/OS・担当工程 | **黒**の点線 |
| 見出し部・スキル評価表 | Excelの罫線のまま |

- 点線の色は**グレーではなく黒**（グレーは薄すぎると却下された）
- 「見出し部の中も点線に」と言われて実装したが直後に撤回された。**見出し部は点線にしない**
- 業務内容の列位置は「業務内容/職務内容/業務経歴」の見出しから自動検出（`CONTENT_COL`）するので、
  列構成が違うスキルシートでも同じ見た目になる

## 検証

過去の別フォーマット4種類（R.Y／Go言語／NW・サーバ／PM・PMO）で再変換し、
色が出る・案件が途中で切れない・文字が枠に接しないことを目視確認済み。

## その他

- 元Excelは横103列（A〜CY）の「Excel方眼紙」形式。横に広いぶん縮小率が大きくなり文字は小さめだが、
  PDFの文字データなので拡大すれば読める。紙で読むならA4横2ページ版という選択肢もある（今回は不要）
- 最終成果物はA4縦2ページ（1ページ目に案件1〜4、2ページ目に案件5）
