# work-repo — Claude Code と仕事を進める作業場

このリポジトリは、あなたが Claude Code と一緒に日常業務を進めるための専用の仕事場（work-repo）です。
応答スタイルと禁止事項の共通ルールはグローバル設定（`~/.claude/CLAUDE.md`）にあり、常にそちらも守ること。
（グローバル設定が未インストールの場合は `setup/README.md` の手順でセットアップする）

## 最重要ルールの再掲（詳細はグローバル設定）

- 鍵（パスワード・API キー等）をファイルに書かない・表示しない・コミットしない
- ファイルの変更はこの work-repo の中だけ。外の原本は触らず「該当タスクの `input/` にコピーしてください」と依頼する
- 外部への送信（メール・Slack）と公開デプロイは、内容を見せて確認してから。公開デプロイは赤木さんのレビュー必須

## タスクの進め方

- 新しい仕事の依頼を受けたら、`init-task` スキルでタスクフォルダを作るところから始める
- タスクは `tasks/YYYYMMDD-タスク名/` 単位で管理し、PLAN / SPEC / TODO / KNOWLEDGE の4ファイルと `input/`（受け取った資料）・`output/`（成果物）を置く
- 詳細な命名・運用ルールは `.claude/rules/task-naming.md` に従う
- 会話の冒頭では、作業対象のタスクフォルダに `HANDOFF.md` があるか確認し、あれば読んで「前回の続きから再開します。{要約}」と報告する
- 作業を中断するときは `handoff` スキルで引き継ぎ書（HANDOFF.md）を残す
- タスクが終わったら `finish-task` スキルで締める（TODO 確認 → 学びの記録 → 保存）
- 繰り返しそうな仕事は、完了後に `make-skill` スキルでスキル化を提案する

## Git の扱い（ユーザーは意識しなくてよい — Claude が全部やる）

ユーザーは git を知らなくてよい。「保存して」「push して」やタスク完了の合図があったら、Claude が以下をまとめて実行する：

1. 作業ブランチを作成してコミットする（ブランチ名: `task/YYYYMMDD-タスク名` 等）
2. GitHub へ push し、PR（プルリクエスト）を作成する（タイトル・本文は日本語で簡潔に）
3. そのまま main へマージする

- main ブランチには直接コミットしない
- コミットの区切り＝「セーブポイント」。作業がある程度進んだら、聞かれなくても「ここまで保存しておきますか？」と提案してよい
- 失敗しても慌てない。「Git のセーブポイントから戻せます」と伝えて対処する
