# タスク運用ルール（tasks/ 配下）

`tasks/` 配下のタスクフォルダを扱うときのルール。

## タスク開始時
1. tasks/ 配下の該当フォルダに移動する
2. input/ フォルダを確認し、添付ファイル（PDF・Excel 等）があれば内容を把握する
3. 以下のファイルを順番に読むこと：
   - HANDOFF.md — 前回セッションの引き継ぎを確認する（存在する場合のみ）
   - PLAN.md — 目的と方針を理解する
   - SPEC.md — 仕様・要件を確認する
   - TODO.md — 現在の進捗を把握する
   - KNOWLEDGE.md — 過去のハマりポイントを確認する

## タスク実行中
- TODO.md の各項目を完了するたびにチェックを入れる
- 新たな知見・ハマりポイントはすぐ KNOWLEDGE.md に追記する

## 入出力ファイルの置き場所（ルート平置き禁止）
- **リポジトリ直下に input/ や output/ を新設したり、ファイルを平置きしたりしない**。タスクの入出力は必ず `tasks/<タスクフォルダ>/input|output/` に置く
- タスクフォルダを作らない単発作業でも、成果物は日付付きタスクフォルダを切ってその `output/` に置く

## タスク完了時
- ユーザーから「終わった」「完了」「OK」等の合図を受けたら、`finish-task` スキルで締める（TODO 全チェック確認 → KNOWLEDGE 追記 → 保存）

## 命名規則

### タスクフォルダ
- ワンショットタスク: `YYYYMMDD-{タスク種別}_{案件名}/`
  - 例: `20260713-契約書レビュー_A社業務委託/`
- 繰り返しタスク: `{周期}-{タスク名}/`
  - 例: `毎月-経費精算/`、`毎週-週報作成/`
- 同種タスクの過去フォルダがあれば、その KNOWLEDGE.md の知見を引き継ぐこと

### アウトプットファイル
- `output/` 配下のファイル名は、タスクフォルダのディレクトリ名をプレフィックスとして付ける
  - 形式: `{ディレクトリ名}_{内容}.md`
  - 例: `20260713-契約書レビュー_A社業務委託_レビュー結果.md`
