## 概要

面談日程調整の一連の手作業（候補日程の読み取り → カレンダー確認 → 調整ブロック登録 → 整形返信 → 確定処理 → 候補者への確定共有）を、Google カレンダー API 連携で自動化するスキル。

運用は2フェーズ：
- **フェーズ1**：候補日程を受け取り、対応可能枠を「面談調整」ブロックで登録＋整形返信を出力
- **フェーズ2**：日程確定後、確定枠＋バッファを登録し、メモ記入・調整ブロック削除・候補者への確定フォーマット出力

## 前提（初回のみセットアップ済みが必要）

Google カレンダー連携は初回セットアップ済み。詳細は `references/setup.md`。要点：
- 秘密情報（`client_secret.json` / `token.json`）と Python 実行環境（venv）は **`~/.config/interview-scheduler/`＝リポジトリ外**。ここには絶対に書かない・コミットしない。
- 実行コマンド（Python）：`~/.config/interview-scheduler/venv/bin/python`
- 対象カレンダー：primary（畑田 真輝 / Asia/Tokyo）
- 未認証・トークン切れなら `scripts/gcal_auth.py` を実行して再認証（本人の通常ブラウザで承認）。

## 確定仕様

- 営業時間：**平日 10:00〜20:00**（土日は対象外）
- 標準所要時間：**1時間**／確定枠は前後 **30分バッファ**
- 突合方針：候補ウィンドウ内で営業時間内かつ空いている連続帯を抽出し、1時間以上入る帯を「○:○○〜○:○○」の幅で返す
- タイトル：調整＝`面談調整｜{クライアント} × {候補者}`、確定＝`【面談 / WEB】{クライアント} × {候補者}`
- **並行調整・重複OK**：面談調整ブロックは「空き時間」表示（transparency=transparent）で作るため、空き計算（freebusy）に影響しない。同じ枠を複数の候補者へ並行して提案でき、面談調整ブロック同士が**重複しても問題ない**（カレンダー上は候補として見える）。空きを消すのは確定面談・通常予定だけ。確定時に、その候補者×クライアントの調整ブロックのみを整理する。

## 手順（フェーズ1：候補日程 → 調整ブロック＋整形返信）

1. **クライアント名・候補者名を確認する**（不明なら聞く）。
2. **候補日程をパースして windows JSON に正規化する**（この整形は Claude が行う）。
   - 形式：`[{"date":"YYYY-MM-DD","start":"HH:MM","end":"HH:MM"}, ...]`
   - 「終日」→ `start` `end` 省略（既定 10:00〜20:00）。「15:00〜開始」など終了なし → `end` 省略（20:00 まで見る）。
   - 日付の表記ゆれ（`7月9日（木）`／`7/1`）は Claude が YYYY-MM-DD に変換。**年が不明なら今日基準で最も近い未来**に補完し、迷う候補は勝手に確定せず確認する。
3. **まず下見（apply=false）で実行**し、対応可能枠を確認する：
   ```bash
   echo '{"client":"A社","candidate":"山田太郎","windows":[...],"apply":false}' \
     | ~/.config/interview-scheduler/venv/bin/python scripts/gcal_register.py
   ```
4. 下見結果（登録予定の枠＋整形返信）をユーザーに見せ、**この内容で登録してよいか確認する**。
5. OK なら **apply=true で再実行**して実際にカレンダー登録する。
6. 出力の「▼整形返信」ブロックをそのままクライアントへの返信に使えるよう提示する（送信自体はしない）。

## 手順（フェーズ2：確定 → ブロック整理 → 候補者へ共有）

1. **確定情報を受け取る**：確定日時、面談ツール、URL、候補者名（＋ふりがな）。
2. **まず下見（apply=false）で実行**する：
   ```bash
   echo '{"client":"A社","candidate":"山田太郎","candidate_kana":"やまだたろう","date":"2026-07-15","start":"14:00","tool":"Google Meet","url":"https://...","apply":false}' \
     | ~/.config/interview-scheduler/venv/bin/python scripts/gcal_confirm.py
   ```
3. 出力される「削除対象の面談調整ブロック一覧」をユーザーに見せ、**確定登録＋バッファ登録＋削除を実行してよいか確認する**（削除は破壊的操作＝必ず対象を列挙して確認）。
4. OK なら **apply=true で再実行**する。確定ブロック（メモ欄フォーマット記入済み）＋前後30分バッファを作成し、該当調整ブロックを全削除する。
5. 出力の「▼候補者への面談確定フォーマット」をそのまま候補者へ共有できるよう提示する（送信自体はしない）。

## スクリプト

- `scripts/gcal_common.py` — 認証・定数・空き枠抽出の共通処理
- `scripts/gcal_match.py` — windows JSON → 整形返信のみ出力（読み取り専用。書き込み不要のときに使う）
- `scripts/gcal_register.py` — フェーズ1：調整ブロック登録＋整形返信（apply で下見/実行を切替）
- `scripts/gcal_confirm.py` — フェーズ2：確定登録＋バッファ＋メモ＋調整ブロック削除＋確定フォーマット（apply で下見/実行を切替）
- `scripts/gcal_auth.py` — 初回・再認証用

## 制約・注意

- **外部送信（メール・Slack・候補者への実送信）はしない**。整形テキストの出力までで、送信はユーザーが手動で行う。
- **カレンダー削除・登録は必ず下見（apply=false）→ 内容確認 → 実行（apply=true）の順**。特に削除は対象を列挙して確認してから。
- **鍵（client_secret / token）は絶対に表示・コミットしない**。保管は `~/.config/interview-scheduler/` のみ。
- パースに迷う候補日程は勝手に確定せず、ユーザーに確認する。
