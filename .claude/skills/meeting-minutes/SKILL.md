## 概要

会議の未整形メモ（チャット貼り付け or input/ のテキスト）を入力に、決まったフォーマットの議事録を Markdown 1枚で生成する。バラバラの箇条書きを「決定事項 / ToDo / 保留・継続確認 / 次回」に仕分け、ToDo は担当と期限を抽出し、曖昧な日付を今日基準で具体日付に補完する。

## 手順

1. **最初に確認する（毎回変わる項目）**
   - 対象の会議メモ（貼り付け or ファイルパス）
   - 保存先タスクフォルダ（無ければ `init-task` で `文書作成_◯◯議事録` を作る。既存の定例フォルダがあればそれを使う）

2. **メモを読み、要素を仕分ける**
   - 会議の日時・参加者をヘッダーにまとめる
   - 各箇条書きを「決定事項 / ToDo / 保留・継続確認 / 次回」に分類する

3. **ToDo の担当・期限を抽出する**
   - 「誰が」「いつまでに」を拾って明記する
   - 担当や期限が読み取れない場合は握りつぶさず「担当: 未定」「期限: 未定」と書く

4. **曖昧な日付を具体日付に補完する**
   - 「火曜まで」「再来週の月曜」「来月」などは、会議日と曜日を確定させてから今日基準で YYYY-MM-DD に直す
   - 会議日そのものが不明なら、メモの日付表記（例: 7/13）＋今年で補完する

5. **output/ に Markdown 1枚で保存し、内容を提示する**
   - 表現の揺れは許容。ユーザーの修正指示があれば反映する

## 出力フォーマット

保存先: `tasks/<タスクフォルダ>/output/<タスクフォルダ名>_<会議日YYYYMMDD>.md`

```markdown
# 定例MTG 議事録

- 日時: YYYY-MM-DD HH:MM〜HH:MM
- 参加者: 氏名・氏名・氏名

## 決定事項
- 決まったこと

## ToDo（担当・期限）
- [ ] 内容（担当: 氏名 / 期限: YYYY-MM-DD）

## 保留・継続確認
- 様子見・保留の項目

## 次回
- YYYY-MM-DD HH:MM
```

## セルフチェック（完了前に必ず確認）

- [ ] 曖昧な日付を、会議日の曜日を確定したうえで今日基準で具体日付に直したか
- [ ] 担当・期限が読み取れない ToDo を「未定」と明示し、握りつぶしていないか
- [ ] 決定事項 / ToDo / 保留 / 次回 の4区分に漏れなく仕分けたか
- [ ] 成果物が output/ の正しい場所・正しい名前で保存されている
