## 概要

任意の MTG（新規商談／社内会議／経営相談／フリーランスとのカジュアル面談 等）を対象に、Google Meet AI 議事録（Docs）や追加メモから**汎用的な議事録 Markdown** を生成する。

出力の基本構成は [MINUTES_TEMPLATE.md](MINUTES_TEMPLATE.md) を遵守する。

## ワークフロー進捗

このチェックリストをコピーして進捗を追跡する:

```
- [ ] ステップ1: 入力ソースの受け取り
- [ ] ステップ2: 各ソースのテキスト化
- [ ] ステップ3: 議事録生成
- [ ] ステップ4: セルフチェック
- [ ] ステップ5: 出力・報告
```

## 前提

- カレントディレクトリはタスクフォルダ配下（`output/` を書き込む）
- 実行環境: Claude Code 上の手動実行
- 判断ロジックは Claude 本体が担う。スクリプトはテキスト抽出のみ

### 依存

- Google Docs API 認証: `~/.config/claude-gdocs-token.json`（既設、`~/.claude/snippets/google_workspace.py` が参照）
- 認証切れ・スコープ不足時は `python3 ~/.claude/snippets/reauth_google.py` を実行

## 入力ソース

| # | ソース | 渡し方 | 必須/任意 |
|---|--------|--------|---------|
| 1 | Google Meet AI 議事録 | Google Docs URL | **必須**（または文字起こしテキストの直接貼り付け） |
| 2 | 追加メモ・文字起こし | テキスト直貼り or ファイルパス | 任意 |
| 3 | 関連レポート・資料 | ローカルパス | 任意 |
| 4 | Google Drive 録画 | Drive URL | 任意（AI議事録で十分な場合はスキップ） |

## 手順

### ステップ1: 入力ソースの受け取り

ユーザーから以下を受け取る（引数で渡されていない場合は対話的に確認）:

1. **Google Docs URL**（必須）
2. **追加ソース**（任意）— メモ・関連レポート等のパス
3. **参加者情報**（任意）— 不明なら議事録から推定

入力が揃ったら、使用ソース・欠落ソースを報告してからステップ2に進む。

### ステップ2: 各ソースのテキスト化

**AI議事録（Google Docs）:**

```bash
python3 - <<'PY'
import sys
sys.path.insert(0, __import__("os").path.expanduser("~/.claude/snippets"))
from google_workspace import get_doc_text
print(get_doc_text('<Google Docs URL or ID>'))
PY
```

- 出力を `input/transcript.txt` に保存（タスクフォルダに `input/` が無ければ作成）
- Gemini 自動議事録は「概要 → 次のステップ → 詳細（タイムスタンプ付き）」の3段構成。**詳細セクションが最も情報密度が高い**

**追加ソース（任意）:**
- テキスト直貼り → そのまま利用
- ファイルパス → Read ツールで読み込み
- **商談準備レポート等が渡された場合、最下部に商談中メモが追記されている運用が多いので、そこを優先の事実ソースとして扱う**

### ステップ3: 議事録生成

[MINUTES_TEMPLATE.md](MINUTES_TEMPLATE.md) のフォーマットに従って Markdown を生成する。

**セクション構成:**

1. **概要** — 日時・参加者・テーマ・結論（一文）
2. **要点まとめ**（振り返り用・**常体**）— テーマ別の小見出し構成
3. **検討事項・ネクストアクション** — チェックリスト形式

**要点まとめのルール:**
- **テーマ別**に小見出しを立てる（時系列で並べない）
- 発言主体を明確にする（例: 「○○様発言」「赤木発言」「両者合意」）
- 事実と所感を分ける。所感は「所感:」プレフィックスで明示
- Gemini 議事録の「次のステップ」は担当者名付きなので、そのままネクストアクションに転記可能

**ネクストアクション・検討事項のルール:**
- チェックリスト形式 `- [ ]` で記述
- 担当者を冒頭に明記（例: `- [ ] **（赤木）資料送付**`）
- 「ネクストアクション」は期限・担当が明確なもの、「検討事項」は意思決定・要確認事項

### ステップ4: セルフチェック

出力前に以下を全チェックする。1つでも違反があれば修正して再生成する:

- [ ] 概要（日時・参加者・テーマ・結論）が揃っているか
- [ ] 要点まとめが**テーマ別**の小見出し構成か（時系列になっていないか）
- [ ] 要点まとめで発言主体が明確か
- [ ] ネクストアクションが担当者付きのチェックリスト形式か
- [ ] Gemini議事録の「次のステップ」がネクストアクションに反映されているか
- [ ] 追加ソース（関連レポート等）内の確定情報が議事録に反映されているか
- [ ] 前置き文（「作成しました」「整理しました」等）が入っていないか

### ステップ5: 出力・報告

1. `output/` ディレクトリが存在しなければ作成する
2. 以下のファイル名で保存: `output/{タスクフォルダ名}_議事録.md`
3. 使用したソース・欠落ソース・出力ファイルパスをユーザーに報告

## 非スコープ

- Google Drive 録画の自動文字起こし（AI議事録の Docs 要約で十分な場合はスキップ）
- CRM への自動書き込み
