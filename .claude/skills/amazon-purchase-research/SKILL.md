## ワークフロー進捗

このチェックリストをコピーして進捗を追跡する:

```
- [ ] ステップ1: ヒアリング

- [ ] ステップ4: 候補ロングリスト化（15〜20件）
- [ ] ステップ5: 要件フィルタ（10件に絞る）
- [ ] ステップ6: 桜チェッカー適用
- [ ] ステップ7: スコアリング & ランキング
```

## 前提

- WebSearch / WebFetch はメインセッションで実行する（サブエージェントには WebFetch 権限が無いことがあり、外部調査が詰まる）
- 桜チェッカー判定で「**危険**」のものは除外する（「警告」は下位配置で残す）
- 候補は新品 / Amazon直販 or 合格メーカー出品を優先、第三者出品のみは除外
- 推測で埋めない。情報が取れない項目は「情報なし」と明記する

## 手順

### ステップ1: ヒアリング

AskUserQuestion を使ってユーザーから以下を順に聞き取る。**項目は対象商品によって増減**するため、AIが適切に質問を組み立てる。

必須:
- **対象商品**（例: USB-Cハブ / 加湿器 / 玄関の棚）
- **予算上限**（「考慮外」も可）
- **必須要件**（サイズ・性能・規格・取付方式・色 など）

任意:
- **スタイル / デザイン**（シンプル / 北欧 / 高級感 等）
- **除外条件**（このメーカーは避けたい等）
- **参考商品 URL**（「○○のような商品が良い」の参考リンク。スタイル・色・素材の参考として扱い、参考品そのものが制約に合わない場合は除外候補に回して代替を提案）
- **件数のデフォルトはTop10**（変更したい場合のみ確認）

#### サイズヒアリングの注意
- 設置場所がある商品（収納・棚・家具系）は **必ず「奥行 / 幅 / 高さ」を区別して聞く**
- 商品側のサイズ表記は「**幅 × 奥行 × 高さ**」順が一般的だが、メーカー・型番により入れ替わるため、候補抽出時に必ず**設置場所の奥行と商品の奥行を一致させる**こと
- 同じ商品でも「縦置き / 横ワイド / 横置き」のバリエーションで奥行と幅が入れ替わる場合がある（例: 無印 PPケース 26×37 と 横ワイド 37×26 は別物）

ヒアリング結果は **後で SPEC.md に書き出す** ためメモしておく。

### ステップ2: スコアリング重みの合意

以下のデフォルト重み案をユーザーに提示し、調整したい場合は対話で変更する。

| 観点 | デフォルト重み | 補足 |
|------|---------------|------|
| 要件適合度 | 高 | 必須条件への合致 |
| 桜チェッカー結果 | 高 | 合格 ＞ 分析不可 ＞ 警告（危険は除外） |
| 価格（予算内） | 中 | 予算超過は除外 |
| デザイン適合 | 中 | スタイル指定への合致 |
| レビュー実評価 | 中 | 桜除外後の星評価・件数 |
| メーカー信頼性 | 低 | 公式サイトあり / Amazon Business 等 |

「このままでよいですか？それとも調整しますか？」と確認。

### ステップ3: タスクフォルダ作成

`02_task/YYYYMMDD-調査・リサーチ_{案件名}/` を作成する（init-task のフォーマット踏襲）。

- 日付は今日の日付
- 案件名はヒアリングした「対象商品」をベースにユーザーに確認（例: `USB-Cハブ購入`）
- サブディレクトリ: `input/` / `output/`
- ファイル: `PLAN.md` / `SPEC.md` / `TODO.md` / `KNOWLEDGE.md`
  - PLAN.md / SPEC.md にはヒアリング結果と合意した重みを記載
  - TODO.md にはステップ4以降のタスクを書き出す

### ステップ4: 候補ロングリスト化（15〜20件）

WebSearch で `site:amazon.co.jp {キーワード} {要件}` 形式の検索を **3〜5パターン展開** する。

検索クエリ例（USB-Cハブの場合）:
- `site:amazon.co.jp USB-C ハブ 4ポート PD100W`
- `site:amazon.co.jp USB Type-C ドッキングステーション 黒`
- `site:amazon.co.jp USB-C hub PD 100W 8000円`

抽出する情報:
- 商品名 / メーカー / 価格 / Amazon URL（ASIN）

**ASINとは**: Amazon の商品ID（`B0`始まりの10桁、URL `/dp/B0XXXXXXXX/` の部分）。桜チェッカーで必須。

### ステップ5: 要件フィルタ（10件に絞る）

必須要件（予算・サイズ・規格 等）を満たさないものを除外し、10件まで絞る。

**サイズ制約「ぎりぎり超過」の扱い**:
- 制約値とのオーバー量が小さい場合（目安: 制約の 3% 以内 / 1cm 以内など）は **自動除外せず警戒点付きで残す**
- 出力時に「○cm 超過 / 設置可否は要実測確認」と明記する
- 厳密に外れる商品（例: 棚奥行26cm に対して奥行50cmの商品）は迷わず除外

**ASINが取れない候補がある場合のリトライ**:
- WebFetch で Amazon ページ取得を試みる
- 5xx エラー時は検索クエリを変えて再探索（メーカー名 + 型番 等）
- 3回試行しても取れないものは除外

**候補数が確保できない場合**:
- 10件未満になった場合はまずステップ4に戻り追加検索（クエリパターンを増やす・別カテゴリ商品も探す）
- それでも揃わなければ **最低5件、要件適合した候補のみで出力する**（無理に件数を埋めない）
- 件数が足りなかった場合は出力 Markdown の冒頭で理由を明記（例: 「サイズ制約が厳しく8件で確定」）

### ステップ6: 桜チェッカー適用

各候補について以下を実行する:

1. WebFetch で `https://sakura-checker.jp/search/{ASIN}/` を取得
2. **判定**（合格 / 警告 / 危険 / 分析不可）を抽出
3. **相対指標**（Amazonと同等 / 少し低い / かなり低い）を抽出
4. **メーカー判定**（合格メーカー / 要注意メーカー / 海外販売 等）を抽出
5. 以下のルールで除外・配置:
   - 「**危険**」判定 → **除外**
   - 「**分析不可** + **要注意メーカー**（海外販売・件数1〜数件）」の組合せ → **除外**（実質的な警告以上相当）
   - 「警告」 → 残すがランキング下位
   - 「分析不可」だが合格メーカー判定あり → 残してメーカー信頼性を参考スコア化
   - 「合格・少し低い／かなり低い」 → 相対指標を一言コメントで明記しつつランキング

### ステップ7: スコアリング & ランキング

ステップ2で合意した重みで総合スコアを算出し、高い順に Top10 を確定する。

各候補に **一言コメント**（評価ポイント・注意点を1行）を付ける。
Top3 には追加で **推奨理由** と **警戒点・注意点** を記載する。

### ステップ8: アウトプット生成

`02_task/{タスクフォルダ}/output/{タスクフォルダ名}_候補一覧.md` に [出力フォーマット](#出力フォーマット) で書き出す。

### ステップ9: ユーザー確認

主要サマリを表示してユーザーに確認:
- Top3 の商品名・順位
- 桜チェッカー除外件数
- 警戒点があれば明示

修正・追加調査があれば対応してから完了報告。

## 注意点・ハマりどころ

### 桜チェッカー
- URL 形式: `https://sakura-checker.jp/search/{ASIN}/`
- レビュー件数が 100 件未満だと「件数不足で分析不可」になりやすい → メーカー単位の合格判定（合格メーカー）が補助指標になる
- 「相場より大幅に安い」「マーケットプレイス出品」「無名メーカー」が組み合わさると警告判定になる傾向
- **「分析不可 + 要注意メーカー（海外販売・件数1）」は実質的な警告以上**として除外する

### Amazon 調査
- Amazon.co.jp の商品ページは WebFetch で 5xx エラーになることがある → 検索クエリを変えて再探索
- URL に残る `smid` パラメータは第三者出品者を指している場合あり。販売元目視確認推奨
- 在庫切れ / 第三者出品のみの商品は除外する
- **同一商品の縦置き / 横ワイドバリエーション**で奥行と幅が入れ替わる場合がある。設置場所の奥行に合った向きの ASIN を選ぶこと（無印 PPケース 26×37 と 横ワイド 37×26 は別 ASIN）

### サイズ制約
- 設置場所がある商品（収納・棚・家具系）は **ヒアリング段階で必ず奥行/幅/高さを区別**して聞く
- ぎりぎり超過（制約の3% 以内 / 1cm 以内）は除外せず警戒点付きで残し、「要実測確認」を出力に明記
- **ユーザー参考品が制約に合わない場合**は、参考品を「除外候補」テーブルに移動し、同シリーズ・同テイストの代替を Top に推奨する

### 出力Markdownの推奨追加要素
- **推奨組み合わせプラン**セクション: 複数並べ・積み重ね前提の商品で意思決定がしやすくなる
- **採寸の最終確認推奨**コメント: ぎりぎり寸法のリスクを明示
- **除外候補テーブル**: ユーザー参考品が不適合だった場合の理由も記載
