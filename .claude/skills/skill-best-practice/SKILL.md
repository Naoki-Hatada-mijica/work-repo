---

## ステップ1: 対象スキルの選択

1. 以下のディレクトリからスキルを走査し、一覧を表示する:
   - **グローバル**: `~/.claude/skills/*/SKILL.md`
   - **プロジェクト**: `.claude/skills/*/SKILL.md`
   - 走査結果が0件の場合は、対象スキルが見つからない旨を伝えて処理を終了する。

2. 一覧の表示形式:

```
## スキル一覧

### グローバル
1. init-task — 新しいタスクフォルダを作成し...

### プロジェクト（{プロジェクト名}）
2. review-contract — SES業務委託基本契約書を...
3. init-task — 新しいタスクフォルダを対話形式で...
```

3. ユーザーに番号またはスキル名で選択を促す
4. 部分一致で候補を絞り込む（例: 「review」→ review-contract）

## ステップ2: スキルの読み込み

1. 対象スキルの SKILL.md を読み込む
2. 同ディレクトリの関連ファイル（.md, .py, .sh等）も一覧確認する
3. 参照されているファイルがあれば読み込む

## ステップ3: 監査の実行

[CHECKLIST.md](CHECKLIST.md) のチェック項目に沿って監査を実行する。

## ステップ4: 監査結果の出力

以下の形式で結果を表示する:

```markdown
# スキル監査結果: {スキル名}

## サマリ
- 合格: X項目
- 要改善: X項目
- 該当なし: X項目

## 要改善項目

### 1. {チェック項目名}（{カテゴリ}）

**現状:** 現在の状態の説明
**改善案:** 具体的な改善提案
**修正例:**
（修正後のコード/テキスト例）

---
（以下繰り返し）

## 合格項目
- {項目名}: OK
```

## ステップ5: 改善の適用

1. ユーザーの承認を得てから修正を適用する
2. 各改善項目について個別に適用可否を確認する
3. SKILL.md を直接編集する
4. 修正後、再度チェックリストを実行して改善を確認する
