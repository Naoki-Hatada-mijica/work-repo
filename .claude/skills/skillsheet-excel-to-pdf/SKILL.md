# Excelスキルシート → 綺麗なPDF

Excelの候補者スキルシートを、**中身はそのまま・見た目を保ったまま**、文字が読みやすいサイズの
A4縦PDFに変換する。Excel標準の「1ページに全部押し込む（極小文字）」問題を解消し、
案件をページ途中で分割せず、複数ページに自然に流し込む。

## 対応フォーマット

案件番号の位置（先頭列/次列）・列数・スキル欄の有無・縦書き見出しの有無などを**自動判定**するので、
フォーマットが多少違うスキルシートでも通る。実際に4種類の別フォーマットで検証済み。以下にも対応：

- **番号のない案件も検出**：案件は「番号」だけでなく「業務内容の見出し（3列以上の結合セルで
  『■』始まり）」でも区切りを検出する。番号なしの個人開発・副業案件なども1まとまりとして扱い、
  その手前で改ページするので途中で切れない（`proj_starts` の (a)番号ベース＋(b)見出しベース）。

- **隠れ行（非表示行）**：Excelで非表示にした行は高さ0で畳む（結合セルに絡んでいても崩れない）。
- **印刷範囲がA1始まりでない**（例：B2:AG85）：印刷範囲の開始位置を尊重して余白を出さない。
- **本文のフォントサイズがバラバラ**：本文（折り返しセル）の中で最も多いサイズを標準とし、
  ずれている本文セルを標準に揃える（例：ある案件の業務内容だけ20pt→他と同じ10ptに統一）。
  短い見出し・タイトルなど別扱いのサイズは維持する。`BODY_SIZE`/`norm_size()` が担当。
- 隠れ列・日付のシリアル値・縦書き見出しなども自動処理。

## ワークフロー

### Step 1: 入力Excelを作業フォルダにコピー
- 対象の `.xlsx` を、該当タスクの `input/` にコピーする（作業場の外の原本は直接触らない）。
- タスクフォルダが無い単発依頼なら、`tasks/YYYYMMDD-スキルシート整形PDF_<候補者名>/` を切り、
  `input/` `output/` を用意する（`.claude/rules/task-naming.md` に従う）。

### Step 2: 変換を実行（HTML生成→PDF化→ダウンロードにコピー→自動で開く）
```bash
python3 .claude/skills/skillsheet-excel-to-pdf/scripts/convert.py \
  "<タスク>/input/<ファイル名>.xlsx" \
  "<タスク>/output"
```
- 第2引数は出力フォルダ（省略時は入力と同じ場所）。第3引数でシート名を指定可（省略時は先頭シート）。
- 出力PDF `<ファイル名>_技術経歴書.pdf` が **output フォルダとダウンロードフォルダの両方**に保存され、
  自動で開く。

### Step 3: 仕上がりを確認
- PyMuPDF等でページ画像を確認し、案件がページ途中で切れていないか・文字サイズが読めるか・
  縦書き見出しが中央に来ているかを見る。必要なら微調整する。

## 仕組み（scripts/xlsx_to_html.py）

Excelの体裁を忠実にHTMLへ写し取り、ヘッドレスChromeでPDF化する。要点：

- **フォントはポイント→ピクセル換算（×96/72）**。Excelのフォントはpt単位。ここを取り違えると
  文字が極小になるので注意（このスキルの一番のハマりどころ）。
- 列幅・行高・罫線・結合セル・塗り・文字揃えを再現。**縦書き（textRotation=255）**は
  `writing-mode: vertical-rl` ＋ `text-align:center` で上下中央に。
- 印刷可能幅に合わせて全体を縮小（狭い表は等倍まで拡大）。行高は `ROW_SCALE=0.9` で軽く詰める。
- **案件は tbody 単位でページ分割禁止**（`KEEP_TOGETHER=True`）。ただし1ページの
  `MAX_KEEP_FRACTION`（既定0.65）を超える大きな案件・塊は分割を許可する。これにより
  「小さい案件は途中で切らない」かつ「大きな塊のせいでページに大きな余白ができない」を両立する。
- 日付が数値（シリアル値）で入っている場合、2000〜2049年相当のものだけ「YYYY年M月」に補正。
- 明らかな誤字を直したい場合のみ、`xlsx_to_html.py` の `CORRECTIONS`（(行,列)→補正文字列）に
  一時的に追記する。既定では中身を一切変えない。

## 注意

- 出力は必ずタスクの `output/` に置く（リポジトリ直下に平置きしない）。
- `~/Downloads` にも自動コピーする（ユーザーはダウンロードで確認する運用のため）。
  古い版が混ざらないよう、毎回同名で上書きコピーされる。
- Word（.docx）はこのスキルでは扱えない。Wordの「ファイル→PDFとして書き出す」で手動対応を案内する。
