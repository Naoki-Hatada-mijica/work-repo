## 概要

タスクの終わりに必ずやるべきこと（進捗確認・学びの記録・保存）を1回で済ませる。
ユーザーは git を意識しなくてよい。保存の仕組みはすべてこのスキルが引き受ける。

## 手順

1. **TODO の完了確認**
   - 対象タスクフォルダの TODO.md を読み、全項目にチェックが入っているか確認する
   - 未完了の項目があれば列挙し、「これらは未完了ですが、締めてよいですか？（残す場合は HANDOFF に引き継ぎます）」と確認する

2. **KNOWLEDGE.md への追記**
   - 今回のタスクでハマった点・工夫した点・次回に活きる学びを KNOWLEDGE.md に追記する
   - 特になければ「特筆事項なし（YYYY-MM-DD 完了）」と1行残す

3. **HANDOFF.md の整理**
   - タスクフォルダに HANDOFF.md が残っていれば、完了済みのため削除する
   - （手順1で未完了項目を残して締める場合は、削除せず未完了項目を「次にやること」として更新する）

4. **GitHub への保存（CLAUDE.md「Git の扱い」に従う）**
   1. 作業ブランチを作成してコミットする（例: `task/20260713-keihi-seisan`。メッセージは日本語で簡潔に）
   2. push して PR を作成する（タイトル・本文は日本語で簡潔に。例:「経費精算タスク完了」）
   3. main へマージする
   - main には直接コミットしない

5. **完了報告＋スキル化の提案**
   - 「タスクを完了し、GitHub に保存しました」と報告する
   - この仕事に繰り返し性がありそうなら、「この仕事、また発生するようなら『スキル化して』と言ってもらえれば、次回から一言で実行できるようにします」と添える

## セルフチェック（完了前に必ず確認）

- [ ] TODO.md の全項目の状態をユーザーと確認した（未完了の握りつぶし禁止）
- [ ] KNOWLEDGE.md に何かしら記録した
- [ ] main への直接コミットをしていない（ブランチ → PR → マージの順）
