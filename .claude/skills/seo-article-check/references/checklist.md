# SEO記事チェックリスト（正本スナップショット）

- 出典: Google Docs「SEO記事手順書・チェックリスト」
  `https://docs.google.com/document/d/1hDgkku4rOhnElaD4cLrd1E9Fuv2ZAfAP9SYdxubwi6w/edit`
- 取得日: 2026-07-07
- 元Docs更新時は `scripts/fetch_article.py` で再取得し、本ファイルとの差分を反映すること。

チェックは2モードに分かれる。**実行時にどちらのモードかを必ず確認する**。
各項目には安定 ID を振ってある。サブエージェントは担当カテゴリの**全 ID に対して
verdict（OK / NG / 判定不能）を必ず返す**こと（項目を落とさないことが本スキルの目的）。

各 ID の `type`:
- `mechanical` … `mechanical_checks.py` が数値判定を出す。サブエージェントはその数値を根拠に verdict を書く（自分で数えない）。
- `judgement` … 文意・品質の判断。サブエージェントが記事本文を読んで verdict を出す。

### 目次
- モード1: 構成チェック … カテゴリ S1 見出し表現 / S2 構成ロジック
- モード2: 本稿チェック … カテゴリ A タイトル / B リード文 / C 構成・見出し / D 本文執筆ルール / E 最終原チェック
- 対象外（判定しない項目）

---

## モード1: 構成チェック（構成案・ロジックツリー段階）

構成案（見出し骨子＋各見出しのメモ）に対するチェック。本文執筆前の段階。

### カテゴリ S1: 見出し表現

| ID | 項目 | type |
|----|------|------|
| S1-1 | 見出しだけで内容がわかりやすい | judgement |
| S1-2 | 各見出しは20〜25字程度 | mechanical |
| S1-3 | 見出しにKWを7〜8割入れている（サブKW・サジェスト含む） | mechanical |
| S1-4 | 共起語・サジェストKWを見出しに記載している | judgement |
| S1-5 | 見出しの階層構造が適切（H2>H3 の入れ子が自然、飛び級がない） | judgement |
| S1-6 | 競合が記載している重要な見出しを網羅している | judgement |

### カテゴリ S2: 構成ロジック

| ID | 項目 | type |
|----|------|------|
| S2-1 | 検索ユーザー・検索意図（why）を踏まえた構成になっている | judgement |
| S2-2 | 実検索の結果からKWを短文化して検索意図を記載している | judgement |
| S2-3 | 顕在ニーズと潜在ニーズを分析して記載している | judgement |
| S2-4 | 競合に勝てる記事のゴールを設定している（独自性の追加にあたりをつけている） | judgement |
| S2-5 | 独自性の根拠（AI補強・ディープリサーチ・実検索）が用意されている | judgement |
| S2-6 | 最初の大見出しで検索意図に答えている | judgement |
| S2-7 | 2つ目以降の大見出しで顕在ニーズ・潜在ニーズに答えている | judgement |
| S2-8 | 各見出し内で記載する内容の結論を記載している | judgement |
| S2-9 | MECEな関係を崩していない | judgement |
| S2-10 | 明らかにノイズと言える競合見出しを削除している | judgement |
| S2-11 | （リライトの場合）見出しの追加だけでなく統廃合も検討している | judgement |

> リライト判定: 構成案に既存記事URL・現順位・既存見出しへの言及があればリライトとみなす。
> 該当が読み取れなければ S2-11 は「判定不能」でよい。

## モード2: 本稿チェック（執筆後の原稿）

完成した記事本文に対するチェック。

### カテゴリ A: タイトル

| ID | 項目 | type |
|----|------|------|
| A-1 | 原則35文字以内 | mechanical |
| A-2 | 見出しの内容と不一致・矛盾がない | judgement |
| A-3 | 読むことで得られるベネフィットを記載している | judgement |
| A-4 | 具体的な数値を使っている | mechanical |
| A-5 | メインKW・サブKWを自然に盛り込んでいる | mechanical |

### カテゴリ B: リード文

| ID | 項目 | type |
|----|------|------|
| B-1 | 型（悩み提示→共感→記事概要）を守っている | judgement |
| B-2 | メインKW・サブKW・共起語を自然に盛り込んでいる | mechanical |
| B-3 | 対策KW・共起語を基本的に左寄せで記載している | judgement |
| B-4 | 記事内容を簡単にまとめている | judgement |
| B-5 | 読者の悩みや疑問を具体的に記載し、共感を誘っている | judgement |
| B-6 | 150〜300字で記載している | mechanical |

### カテゴリ C: 構成・見出し（本稿）

| ID | 項目 | type |
|----|------|------|
| C-1 | 見出しだけで内容がわかりやすい | judgement |
| C-2 | 各見出しは20〜25字程度 | mechanical |
| C-3 | 見出しにKWを7〜8割入れている | mechanical |
| C-4 | 見出しの階層構造が適切 | judgement |
| C-5 | 最初の大見出しで検索意図に答えている | judgement |
| C-6 | 2つ目以降の大見出しで顕在・潜在ニーズに答えている | judgement |
| C-7 | 各見出し内で結論を記載している | judgement |
| C-8 | MECEな関係を崩していない | judgement |

### カテゴリ D: 本文執筆ルール

| ID | 項目 | type |
|----|------|------|
| D-1 | PREP等の型を守って平易な表現で執筆している | judgement |
| D-2 | 1文の長さは60字程度（80字超は要修正） | mechanical |
| D-3 | 80〜120字で改行している | mechanical |
| D-4 | 引用・参考・参照のルールを守っている | judgement |
| D-5 | 具体的な他社サービス名を書いていない | judgement |
| D-6 | エージェントサービスに触れるならCTAを試みている | judgement |
| D-7 | （可能な限り）独自性を追加している（画像・解決策など） | judgement |
| D-8 | （可能なら）関連記事をルールに則って挙げている | judgement |

### カテゴリ E: 最終原チェック

| ID | 項目 | type |
|----|------|------|
| E-1 | 文章がねじれていない | judgement |
| E-2 | 結論から先に書くPREPなどの型になっている | judgement |
| E-3 | 「てにをは」など助詞が適切に使えている | judgement |
| E-4 | 句読点の位置が適切で多すぎない | judgement |
| E-5 | 誤字脱字・表記ゆれがない | judgement |
| E-6 | 声に出して読んで違和感がない | judgement |
| E-7 | 根拠なく強い表現が使われていない（例:「絶対年収があがる！」） | judgement |
| E-8 | 対策キーワードを段落ごとに入れられている | mechanical |

## 対象外（本スキルでは判定しない）

記事テキストからは検証できないため、レポートに「対象外」として明記する。

- **リサーチ工程全般**: 競合サイト調査（SEO META in 1 CLICK）、サジェストKW調査、共起語調査、
  SERP分析（バーティカル/ユニバーサル・PAA・関連KW・ペルソナのシート転記）
- **コピペチェック**: 外部コピペチェックツールでの実施
- **入稿時**: 「赤木 宏志 | mijica CEO から一言」へのXリンク付与
- **入稿後**: ラッコキーワードでの順位取得・履歴反映

> これらは「チェックリストに存在するが本スキルの守備範囲外」であることをレポート冒頭で
> 明示し、ライター/入稿担当が別途担保する前提とする。
