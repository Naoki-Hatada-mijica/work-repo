## 目的

当社（株式会社mijica、フリーランス特化のITエージェント／SES事業）が、パートナー企業（BP＝ビジネスパートナー）から紹介された案件をフリーランスへ提案する際に使う。
ユーザーがターミナルに貼り付けた**未整形の案件情報**を、提案用の**指定フォーマット（FMT）**へ正確に整形して返す。

## 入力

- ユーザーがターミナルに直接貼り付けた、未整形の BP 案件情報（自由形式テキスト）。
- **1回につき1案件のみ**処理する（複数案件のベタ貼りは想定しない。複数貼られた場合は1件目のみ整形し、その旨を一言添える）。
- 案件情報が貼られず曖昧な依頼だった場合は、「整形したい案件情報を貼ってください」と促す。

## 変換ルール

1. **支払いサイト_締め日** は必ず「40日サイト (当月末締め翌々月10日支払い)」に固定する（入力に別記載があっても固定）。
2. **不明な項目は行ごと削除する**。情報が無い項目に `●` を残さず、その `■項目：` 行ごと出力から削る。**情報が無い項目を推測で埋めない**（削除する）。
   - **例外（コア項目・常に残す）**: `{案件名}`・`■案件の内容`・`■必須スキル`・`■単価`・`■支払いサイト_締め日`。これらは不明時のみ `●` を残す。
   - `■希望する年齢層`・`■外国籍` は**情報の有無にかかわらず常に削除**する（候補者へ共有しない項目のため）。
3. **開発環境** の情報（言語/FW/インフラ/その他のいずれか）が**一切無ければ**、`■開発環境` セクションを**見出しごと丸ごと削除**する。1つでも情報があればセクションを残し、情報が無いサブ項目（言語/FW/インフラ/その他）はその行を削除する。
4. **稼働日数** は、入力に特段の言及（週4日可・週3日等）が無ければ `■稼働日数：週5日` とする（既定は週5日）。
5. **稼働形態** は原文のニュアンスを保持する（「基本リモート」「フルリモート」「常駐」等、書かれたまま）。勝手に「リモート併用」等へ言い換えない。
   - 出社可能性・勤務地（地方在住可など）に関する補足は、**稼働形態の括弧書きで追記**する。例: `■稼働形態：基本リモート（原則リモートだが、PJ状況に応じて出社をお願いする可能性あり。地方在住の方も可）`。これらは `■備考` には重複させない。
6. **「弊社」は「元請」に置換する**。BP案件では入力中の「弊社」は紹介元の上位会社（＝BP＝元請）を指すため、出力では「弊社」を「元請」に置き換える。
   - **例外**: 後述の固定文言 `■商流：エンド→元請→弊社` の「弊社」は当社（mijica）自身を指すため置換しない（常にこの文言のまま）。
6.5. **`■商流：エンド→元請→弊社` は固定文言として常に挿入する**。`■商談` 行の直下に、入力の有無・内容にかかわらず**この文言のまま**1行を挟む（商流の記載が入力に無くても削除しない）。
7. 各スキル・条件の箇条書きは適切に改行して整理する（必須スキル・歓迎スキルは1項目ずつ `・` で改行）。
   - `■案件の内容` など**長文の項目は、1行が長くなりすぎないよう文の切れ目・句点（。）で改行を入れる**。1文ごと、または意味の区切りごとに改行し、読みやすく整える。
8. **単価はスキル見合いレンジへ変換する**（下記「単価変換ロジック」を厳守）。
9. 提案資料やドキュメントファイルとして出力せず、**チャットベース（コードブロック形式）**で返信する。
10. 出力 FMT を**記号・見出し・固定文言を含めて正確に**出力する（不明項目を削除した行を除く）。

## 単価変換ロジック（ルール8の詳細）

提示単価から **下限側は15万円、上限側は10万円** を差し引いてレンジ化し、
`スキル見合い（{下限} ~ {上限}万/税別）` の形式で記載する。

- **単一値で提示**: `X万` → 下限 = X−15、上限 = X−10
  - 例: 90万 → `スキル見合い（75 ~ 80万/税別）`
  - 例: 80万 → `スキル見合い（65 ~ 70万/税別）`
- **レンジで提示**: `下限L 〜 上限H 万` → 下限 = L−15、上限 = H−10
  - 例: 90〜100万 → `スキル見合い（75 ~ 90万/税別）`
  - 例: 70〜80万 → `スキル見合い（55 ~ 70万/税別）`
- **税込/税別の区別はしない**。提示値をそのまま差し引きに使う（税別換算 ÷1.1 はしない）。FMT 上の `/税別` 表記は固定で常に維持する。
- **単価の記載が無い場合**は計算せず `スキル見合い（● ~ ●万/税別）` のまま残す。

## 出力フォーマット（FMT）

以下は**完全形のスケルトン**。`{案件名}` と各 `●` を入力に応じて置換し、**情報が無い項目は行ごと削除**（ルール2）、**`■希望する年齢層`・`■外国籍` は常に削除**した上で出力する。
出力は必ずコードブロック（```）で囲む。

```
-------------------------------
{案件名}
-------------------------------
■案件の内容：
●

■必須スキル：
・●
・●

■歓迎スキル：
・●

■開発環境：
言語：●
FW：●
インフラ：●
その他：●

■稼働日数：週5日,週4日,週3日
■支払いサイト_締め日：40日サイト (当月末締め翌々月10日支払い)
■商談：●回 (オンライン)
■商流：エンド→元請→弊社
■募集人数：●人
■開始時期：●月
■稼働形態：フルリモート/リモート併用/常駐
■単価：スキル見合い（● ~ ●万/税別）
■精算幅：●h - ●h
■作業開始/終了時間の目安：●:00-●:00
■希望する年齢層：●~●歳
■外国籍：可,不可
■PC貸与：●
■備考：●
-------------------------------
```

### FMT に関する補足
- `■稼働形態` のように**選択肢が併記されている項目**は、入力で特定できればその値だけに絞る。特定できなければ（＝不明なら）ルール2に従い**行ごと削除**する。
- `■外国籍` は入力で可/不可が特定できても**常に削除**する（`■希望する年齢層` と同じ扱い・ルール2）。
- `■稼働日数` は既定が週5日（ルール4）。入力に別条件があればその値に絞る。
- `{案件名}` は入力から抽出する。明確な案件名が無ければ案件内容を端的に表すタイトルを置く（伏せ字 `●` ではなくタイトルを入れる）。

## 出力前セルフチェック

出力する前に、以下を必ず確認する:

- [ ] `■支払いサイト_締め日` が「40日サイト (当月末締め翌々月10日支払い)」固定になっているか
- [ ] 開発環境の情報が無い場合に `■開発環境` セクションを丸ごと削除したか（1つでもあれば残し、不明サブ項目の行は削除）
- [ ] 不明な項目を `●` で残さず**行ごと削除**したか（コア項目＝案件名/案件の内容/必須スキル/単価/支払いサイトのみ `●` 可）
- [ ] `■希望する年齢層`・`■外国籍` を削除したか（いずれも常に削除）
- [ ] `■稼働日数` が既定の週5日になっているか（別条件があればその値）
- [ ] `■稼働形態` のニュアンスを原文どおり保持し、出社可能性・勤務地補足を括弧書きで追記したか
- [ ] 入力中の「弊社」を「元請」に置換したか（ただし固定文言 `■商流：エンド→元請→弊社` は除く）
- [ ] `■商談` 行の直下に固定文言 `■商流：エンド→元請→弊社` を挿入したか
- [ ] `■案件の内容` など長文項目を句点・文の切れ目で改行し、1行が長すぎないか
- [ ] 単価の差し引きが正しいか（下限 −15万 / 上限 −10万、`/税別` 表記維持）
- [ ] FMT の区切り線・見出し記号（`■` `・`）が欠けていないか
- [ ] コードブロック（```）形式で返しているか

## 出力後

- 整形結果（コードブロック）に続けて、**コア項目で `●` のまま残した／入力の矛盾・要確認点があればリスト**で簡潔に提示し、「不足情報があれば追記して再整形できます」と添える。
- 単価を差し引き変換した場合は、計算根拠（例: 提示90万 → 75〜80万）を1行で補足する。
- **最後に必ず「pbcopy しますか？」と確認する**。承認されたら、整形結果（コードブロックの中身＝区切り線から区切り線までの本文）をクリップボードへコピーする。
  - コピー方法: 整形結果テキストを `pbcopy` に渡す（例: `cat <<'EOF' | pbcopy ... EOF`、または一時ファイル経由で `pbcopy < file`）。コードブロックのバッククォート（```）自体は含めない。
  - 再整形のたびに最新の結果で再確認する。
