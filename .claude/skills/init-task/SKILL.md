name: init-task
description: 新しいタスクフォルダを対話形式で作成し、PLAN.md/SPEC.md/TODO.md/KNOWLEDGE.mdを初期化する
---

## 手順

1. ユーザーに以下を順番に確認する（すべてAskUserQuestionを使用）：
   1. 「繰り返しタスクですか？」→ はい / いいえ
   2. （はいの場合）「どの程度の周期ですか？」→ 毎日 / 毎週 / 毎月
   3. （いいえの場合）「タスク種別を教えてください」→ 以下の選択肢＋自由入力（Other）
      - MTG準備 — 会議・商談の準備
      - 調査・リサーチ — 情報収集・市場調査
      - 契約書レビュー — 契約書の確認・レビュー
      - 文書作成 — 資料・レポート・ヒアリングシート等の作成
      - スキル作成・修正 — Claudeスキルの新規作成・改善
      - 新規商談準備 — クライアントとの新規商談準備
      - シミュレーション — 数値シミュレーション・試算
      - 技術検証 — 技術的な検証・PoC
   4. 「案件名を教えてください」→ 自由入力（AskUserQuestionは使わず、ターミナル上で入力を待つ）

2. **git worktree を作成する**（並行セッションの作業ツリー衝突を避けるため）
   - ユーザーが「以前のタスクの続き」と明示した場合はこのステップをスキップし、既存ブランチで継続する
   - 案件名から簡潔な英語ケバブケースの `<task-id>` を生成する（日本語案件名の場合も英語識別子を付ける。例: `契約書レビュー_A社` → `contract-review-a`）
   - メインブランチの最新から worktree を切る（prefix は内容に応じて `feat/` `fix/` `docs/`、日常の文書・調査系は既定で `chore/`）：
     ```bash
     git fetch origin
     git worktree add .claude/worktrees/<task-id> -b chore/<task-id> origin/main
     cd .claude/worktrees/<task-id>
     ```
     - リモートが無い／到達不可の場合は `main`（ローカル最新）から切る
   - 以降のフォルダ・ファイル作成は、すべてこの worktree ディレクトリ内で行う

3. worktree 内の tasks/ 配下にフォルダを作成する
   - 繰り返しタスクの場合: `{周期}-{案件名}/`（例: `毎月-経費精算/`）
   - ワンショットタスクの場合: `YYYYMMDD-{タスク種別}_{案件名}/`（例: `20260316-契約書レビュー_A社業務委託/`）
   - 日付は今日の日付を使う

4. 以下のディレクトリを作成する：
   - input/ — 添付ファイル・PDF等の入力素材の格納先
   - output/ — 成果物の格納先

5. 以下の4ファイルを作成する：

### PLAN.md
1. 空のテンプレートで作成する
2. `code` コマンドでVS Codeで開く
3. ユーザーに「VS Code上でPLAN.mdを編集してください。音声入力・コピペ・手書きなど自由に記載してください。完了したら教えてください」と伝え、完了の返答を待つ
4. 完了後、PLAN.mdを読み取る
5. 内容に不明点・不足があればターミナル上で質問し、回答をPLAN.mdに追記する
6. 質問がなければ、または質問完了後、PLAN.mdの内容を構造化して書き直す

### SPEC.md
PLAN.mdの内容をもとにユーザーと壁打ちし、
要件・観点・制約を整理して記載する。

### TODO.md
SPEC.mdからタスクを分解し、チェックリスト形式で記載する。

```markdown
- [ ] タスク1
- [ ] タスク2
```

### KNOWLEDGE.md
空のテンプレートで作成する。

```markdown
# Knowledge

過去の学びや注意点をここに記録する。
```

6. 同種の過去タスクが tasks/ にあれば、その KNOWLEDGE.md を読み込み、
   新しいタスクの KNOWLEDGE.md に引き継ぐ

7. 完了報告時に、作成した worktree のパス（`.claude/worktrees/<task-id>`）と
   ブランチ名をユーザーに伝える
