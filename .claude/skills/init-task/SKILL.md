## 手順

1. ユーザーに以下を順番に確認する（すべて AskUserQuestion を使用）：
   1. 「繰り返しタスクですか？」→ はい / いいえ
   2. （はいの場合）「どの程度の周期ですか？」→ 毎日 / 毎週 / 毎月
   3. （いいえの場合）「タスク種別を教えてください」→ 以下の選択肢＋自由入力（Other）
      - MTG準備 — 会議・商談の準備
      - 調査・リサーチ — 情報収集・市場調査
      - 契約書レビュー — 契約書の確認・レビュー
      - 文書作成 — 資料・レポート・ヒアリングシート等の作成
      - データ整理・集計 — Excel/CSV の整理・集計・チェック
      - 新規商談準備 — クライアントとの新規商談準備
      - 定型業務 — 毎回ほぼ同じ手順で行う事務作業
      - 技術検証 — ツール・仕組みの検証
   4. 「案件名を教えてください」→ 自由入力（AskUserQuestion は使わず、チャット上で入力を待つ）

2. tasks/ 配下にフォルダを作成する
   - 繰り返しタスクの場合: `{周期}-{案件名}/`（例: `毎月-経費精算/`）
   - ワンショットタスクの場合: `YYYYMMDD-{タスク種別}_{案件名}/`（例: `20260713-契約書レビュー_A社業務委託/`）
   - 日付は今日の日付を使う

   > **命名・配置ルールの正本は `.claude/rules/task-naming.md`**（フォルダ名・`output/` ファイル名のプレフィックス・ルート平置き禁止等）。このルールは `tasks/` 配下のファイルを Read した時に自動ロードされる。新規作成だけの場合は自動発火しないため、本手順の命名規則に沿って作成すること（不安なら既存の `tasks/` フォルダを1つ Read してルールを呼び出す）。

3. 以下のディレクトリを作成する：
   - input/ — 添付ファイル・PDF 等の入力素材の格納先
   - output/ — 成果物の格納先

4. 以下の4ファイルを作成する：

### PLAN.md
1. ユーザーに「やりたいことを、思いつくまま雑に話してください（箇条書き・断片でOK。きれいにするのはこちらの仕事です）」と伝え、返答を待つ
2. 返答をもとに PLAN.md を作成する（目的・背景・ゴールのイメージを整理して書く）
3. 内容に不明点・不足があればチャットで質問し、回答を PLAN.md に反映する

### SPEC.md
PLAN.md の内容をもとにユーザーと壁打ちし、要件・観点・制約を整理して記載する。
質問は一度に多くても3つまでにし、専門用語を避ける。

### TODO.md
SPEC.md からタスクを分解し、チェックリスト形式で記載する。

```markdown
- [ ] タスク1
- [ ] タスク2
```

### KNOWLEDGE.md
空のテンプレートで作成する。

```markdown
# Knowledge

過去の学びや注意点をここに記録する。
```

5. 同種の過去タスクが tasks/ にあれば、その KNOWLEDGE.md を読み込み、
   新しいタスクの KNOWLEDGE.md に引き継ぐ

6. 完了報告として、作成したフォルダのパスと「この後は TODO.md の上から順に進めます」を伝え、
   最初の TODO 項目に着手してよいか確認する
