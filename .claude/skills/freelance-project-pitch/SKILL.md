## 概要

フリーランスエージェントが候補者に案件を電話・面談で紹介する際の「そのまま読める口頭説明原稿」を生成する。

ゴールは「優秀な人事担当が自社の魅力を候補者に伝える」ような原稿。業界の課題説明やテンプレ的な前置きは不要。企業のこと、プロダクトのこと、ポジションのことを、事実ベースで魅力的に伝える。

入力は未整理・不足・重複がある前提で処理する。

## ワークフロー

### Step 1: 入力の受け取り

ユーザーから以下を受け取る（形式不問：箇条書き・文章・コピペ混在可）:

- **企業名**（必須）
- **企業URL**（必須）
- **案件募集内容**（必須）
- **プロダクト情報**（あれば）

### Step 2: 企業情報の補完

企業URLをWebFetchで取得し、以下を把握する:

- 事業内容・提供サービスの具体的な内容
- 対象顧客・導入実績
- 成長フェーズ・資金調達・グループ企業などの定量的な魅力
- プロダクトの特徴・強み

WebFetchで取得できた情報は原稿に活用するが、取得できなかった情報は推測せず、ユーザー提供の情報のみで構成する。

### Step 3: 原稿の素材整理（出力しない、内部準備）

原稿を書く前に、以下を整理する:

#### A. 企業の紹介
- どんな会社か（事業領域、規模感、グループ企業、拠点展開など）
- 候補者が「しっかりした会社だな」「勢いがあるな」と感じる事実

#### B. プロダクトの紹介
- どんなプロダクトを開発しているか
- そのプロダクトの何が面白いか・どんな価値があるか
- 成長フェーズ（導入状況、新規開発中、事業拡大中など）

#### C. ポジション・求める人材
- 今回どういう役割で入るのか（チーム内の立ち位置、期待されていること）
- どういう経験・人物像を求めているか

### Step 4: 原稿の生成

#### 構成（3パート構成・一続きの文章で）

1. **企業紹介** — どういった会社か。事実で信頼感や勢いを伝える（グループ企業、拠点、実績など）
2. **プロダクト紹介** — どんなプロダクトを開発しているか。何が面白いか、どんなフェーズか
3. **ポジション・求める人材** — 今回どういう役割での募集か。何を期待されていて、どんな方を求めているか

この3つを自然に繋げて一続きの文章にする。パートの切れ目が目立たないよう、流れるように繋ぐ。

#### 魅力の伝え方

- **事実で語る**: 「すごい会社」とは言わない。グループイン先、導入実績、拠点数、新規プロダクト数などの具体的事実を出せば、聞き手が自然に魅力を感じる
- **プロダクトの面白さ**: そのプロダクトが「何を実現するか」を候補者が想像できるように伝える。単なる業種ラベル（「不動産テック」等）ではなく、具体的に何をするプロダクトかが分かるように
- **ポジションの特別感**: 募集要項の職種名をそのまま言うのではなく、どういう立場でどんなことを任されるのかが伝わるように
- **業界課題の説明は不要**: 「〇〇業界はまだアナログで〜」のような前置きは入れない。プロダクトや会社の説明の中で自然に伝わればよい

#### 絶対に守るルール

- **技術スタックは一切出さない**（別途資料で見せるため）
- **条件面（単価・稼働日数・勤務場所・期間・リモート可否など）は一切出さない**
- **不明な情報は補完・推測しない**
- 重複表現はまとめる
- 説明しづらい固有名詞は、簡単な言い換えを添える

#### 文体・トーン

- 話し言葉（電話・面談で読み上げる前提）
- 優秀な人事担当が自社の魅力を自然に伝えるトーン。押し売りでも淡々ともしない
- 「〜ようなポジションです」「〜フェーズになってます」「〜を期待されています」のような、エージェントが自然に話せる語尾
- 一文が長くなりすぎないようにする

#### 文字数

250〜350文字程度

### Step 5: 出力

**出力形式は厳守:**

- 出力は必ずコードブロック（```）で囲んだ本文のみ
- コードブロックの外には一切テキストを出力しない
- 箇条書き、見出し、装飾は使用しない
- 原稿本文だけを返す

## 出力例

### 例1

入力: ミチビク株式会社 / https://michibiku.co.jp/ / リードエンジニア募集、取締役会DX SaaS、マネーフォワードグループ、CTOの右腕

```
取締役会の運営効率化と議論の見える化を実現するDX SaaS「michibiku」を開発しているミチビク社の案件です。直近マネーフォワードにグループジョインして事業を伸ばしていて、上場企業を中心に導入もかなり進んでいるフェーズになってます。今後はグループのシナジーを活かして取締役会にとどまらず経営の周辺領域にも展開していく構想で、AIを活用した経営判断の高度化にも取り組み始めています。今回はCTOの右腕として長期的に並走し、プロダクト成長を支えていただける「リードエンジニア」を募集していて、ハンズオンでの開発牽引はもちろん、技術的メンタリングなどEM的な役割も担い、チーム全体の底上げに貢献していただくことを期待されているようなポジションです。
```

### 例2

入力: 株式会社TERASS / https://terass.com/ / モバイルエンジニア募集、不動産エージェント向けプラットフォーム、年間2〜3新規プロダクト、技術選定から関与

```
不動産売買の領域で、優秀なエージェントがもっと効率的かつ自由に働けるプラットフォームを展開しているTERASS社の案件です。物件情報の管理や住宅ローンの比較、価格査定といったエージェント業務を支える複数のプロダクトを開発していて、東京・大阪・名古屋・福岡・札幌と全国に拠点を広げながら、年間2〜3の新規プロダクトを出しているフェーズになってます。今回はモバイルアプリの新規開発プロジェクトで、技術選定やアーキテクチャ設計の段階から関わっていただけるポジションです。仕様通りに作るだけではなく、現場で本当に使われるモバイル体験をゼロから形にしていく役割を期待されています。
```

### 例3

入力: アイザック株式会社 / https://aisaac.jp/ / マーケットプレイス事業エンジニア、自己資本100%、グループ複数事業、二次流通特化型マーケットプレイス、共通基盤あり、立ち上げフェーズ

```
マッチング・AI・ヘルステック・アートなど幅広い領域でグループ会社を持ち、自己資本100%で複数の事業を展開しているアイザック社の案件です。今回はグループの新規事業として、二次流通の特化型マーケットプレイスを立ち上げるプロジェクトになります。ジャンルごとに最適化したマーケットプレイスを、決済やオペレーションの共通基盤を活用しながら次々と立ち上げていく構想で、まさに今その立ち上げフェーズにあります。ポジションとしては設計から実装まで一気通貫で関わりながら、何を作るかの企画段階からプロダクトを伸ばすところまで裁量を持って担っていただく形です。フルスタックでの開発経験があり、事業視点を持って自走できる方を求めています。
```
