## 概要

取引先クライアントへ年2回送る季節のご挨拶メールを作成するスキル。
差出人は「株式会社mijica 畑田」。1通で「①季節のご挨拶 ②長期休業のご案内 ③お中元／お歳暮の送り状 ④直接訪問できないお詫び」を兼ねる。
入力＝その回の休業日程（＋任意で送付先リスト）。出力＝メール版とSlack版の文面（コピペ可）。メール送信自体はユーザーが手動で行う。

## 手順

1. **どちらの回か確認する**：夏版（お中元・夏期休業）／冬版（お歳暮・年末年始休業）
2. **休業日程を確認する**：休業期間（開始日〜終了日）。ここから以下を算出：
   - 休業前最終営業日（休業開始日の前営業日。土日を除く。`date -j` で曜日を確認）
   - 営業開始日（休業終了日の翌営業日）
3. **送付タイミングを案内する**（`references/timing.md` を参照）
   - 夏：本命は7月上旬〜中旬（関東寄り7/10前後・関西寄り7月中旬）、任意で休業直前にリマインド
   - 冬：本命は12月上旬（12/1〜12/10）、任意で最終営業日の週にリマインド
4. **メール版を作成する**：`references/template.md` の該当版をベースに、件名・宛名差し込み欄・罫線の休業案内ブロック・お中元／お歳暮の送り状・お詫び・季節の結びを組み立てる
5. **Slack版を作成する**：同趣旨を柔らかめトーンに。罫線ブロックは簡易な箇条書きにする
6. **（送付先リストがある場合）**：会社名・担当者名を差し込んだ個別文面を一括生成する
7. 出力を確認してもらう（送信はユーザーが手動で対応）

## 出力フォーマット

- 置き場所：`tasks/年2回-季節のご挨拶メール/output/`
- ファイル名：`年2回-季節のご挨拶メール_{夏版|冬版}_メール文面.md`
- 中身：件名／メール本文／Slack用文面。宛名は `株式会社●●／●●様` の差し込み欄にする

## セルフチェック（完了前に必ず確認）

- [ ] 休業前最終営業日・営業開始日の曜日が正しい（土日を挟む場合に注意。`date -j -f "%Y-%m-%d"` で検算）
- [ ] 夏はお中元・「猛暑の折ご自愛」、冬はお歳暮・「良いお年を／ご自愛」と、季節に応じた語になっている
- [ ] お中元／お歳暮の送付案内と、直接訪問できないお詫びが入っている
- [ ] 送付タイミングの推奨を伝えている
- [ ] メール送信は勝手に行わず、ユーザーの手動送付を前提にしている
- [ ] 送付先リストは個人情報。output/ の文面にベタ書きした場合はコミットしない配慮をする
- [ ] 成果物が output/ の正しい場所・正しい名前で保存されている
