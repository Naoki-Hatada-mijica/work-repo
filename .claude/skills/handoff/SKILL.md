## 概要

現在の作業状況を `HANDOFF.md` に保存し、次の会話で「前回の続き」から再開できるようにする。
会話の中身はリセットすると消えるが、このファイルに書いたことは消えない。

## 保存先の判断（上から順に、最初に一致したものを使う）

1. **タスクフォルダ配下で作業している**（`tasks/<タスク>/`）
   → そのタスクフォルダ直下の `HANDOFF.md`（既存は上書き）
2. **いずれでもない**
   → リポジトリのルート直下の `HANDOFF.md`（既存は上書き）

## 実行手順

1. 現在の作業ディレクトリを確認し、上記ルールで保存先を判断する
2. 保存先に `HANDOFF.md` を作成する（既存は上書き）
3. 以下のテンプレートに従い、現在までのチャット履歴から具体的な内容を記入して保存する
   - 単なる空テンプレートのまま保存してはいけない

```markdown
# HANDOFF — {タスク名}

最終更新: {YYYY-MM-DD}

## 要約
1〜2文で現在の状態を要約

## 今回のセッションでやったこと
- 完了した作業を具体的に記載

## 現在の状態
- 進捗状況（TODO.md のどこまで完了したか等）

## 次にやること
1. 最優先のアクション
2. 次のアクション

## 注意点・ブロッカー
- 注意すべき事項（なければ「なし」と記載）
```

4. 保存後、「引き継ぎ書を保存しました。GitHub にも保存（push）しておきますか？」と確認する
   - はいの場合、CLAUDE.md の「Git の扱い」に従って保存する（ブランチ作成 → コミット → PR 作成 → マージ）

## 完了報告

保存先と要約をユーザーに報告し、「次の会話ではこの引き継ぎ書から再開できます」と伝える。
