## 概要

現在の作業状況を `HANDOFF.md` に保存し、次のセッションで再開できるようにする。

## 実行手順

1. 現在の作業ディレクトリを確認する
2. 引き継ぎ対象のタスクディレクトリまたはプロジェクトルートを判断する
3. 対象ディレクトリ直下に `HANDOFF.md` を作成する（既存の場合は上書き）
4. 以下のテンプレートに従い、現在までのチャット履歴から具体的な内容を記入して保存する
   - 単なる空テンプレートのまま保存してはいけない

```markdown
# HANDOFF — {タスク名}

最終更新: {YYYY-MM-DD}

## 要約
1〜2文で現在の状態を要約

## 今回のセッションでやったこと
- 完了した作業を具体的に記載

## 現在の状態
- 進捗状況（TODO.mdのどこまで完了したか等）

## 次にやること
1. 最優先のアクション
2. 次のアクション

## 注意点・ブロッカー
- 注意すべき事項（なければ「なし」と記載）
```

## 保存先の判断

- `tasks/<task-name>/` 配下で作業している場合: そのタスクディレクトリ直下の `HANDOFF.md`
- `.spec/` を持つプロジェクトの場合: プロジェクトルートの `HANDOFF.md`
- 判断できない場合: 現在の git ルート直下の `HANDOFF.md`

## 完了報告

作成後、保存先と要約をユーザーに報告する。
