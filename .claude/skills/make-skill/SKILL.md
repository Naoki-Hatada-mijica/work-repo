## 概要

一度うまくいった仕事のやり方をスキル（手順書）として保存する。
次回からはユーザーが一言（例:「経費精算やって」）言うだけで、同じ手順・同じ品質で実行できるようになる。

## 手順

1. **対象タスクの特定**
   - 直近に完了した（または今の会話で進めていた）タスクを対象とする
   - どのタスクか曖昧な場合は「どの仕事をスキル化しますか？」と確認する

2. **手順の抽出**
   - 対象タスクの PLAN.md / SPEC.md / TODO.md / KNOWLEDGE.md と今回の会話の流れから、
     繰り返し可能な手順を抽出する
   - 「今回だけの事情」（特定の日付・特定の相手先など）と「毎回共通の手順」を区別し、
     毎回変わる部分は「ユーザーに確認する項目」として整理する

3. **スキル名と発動キーワードの確認**
   - スキル名（英語ケバブケース。例: `expense-report`）の案を提示して確認する
   - 発動キーワード（ユーザーが普段言いそうな言い方。例:「経費精算やって」）を確認する

4. **SKILL.md の作成**
   - `.claude/skills/<スキル名>/SKILL.md` を以下のテンプレートで作成する

```markdown
---
name: {スキル名（英語ケバブケース）}
description: {何をするスキルか1文}。ユーザーが「{発動キーワード1}」「{発動キーワード2}」と言った場合に使用する。
---

## 概要

{このスキルが何を入力に、何を出力するか。2〜3行}

## 手順

1. {ステップ1 — ユーザーに確認する項目があれば最初にまとめて確認}
2. {ステップ2}
3. ...

## 出力フォーマット

{成果物の型・置き場所（tasks/<タスクフォルダ>/output/ 配下）・ファイル命名}

## セルフチェック（完了前に必ず確認）

- [ ] {今回ハマった点・間違えやすい点をチェック項目にする}
- [ ] 成果物が output/ の正しい場所・正しい名前で保存されている
```

5. **完了報告**
   - 「スキル `{スキル名}` を保存しました。次回からは『{発動キーワード}』と言うだけで、この手順で実行します」と報告する
   - CLAUDE.md の「Git の扱い」に従って保存（push）するか確認する

## 注意

- description には発動キーワードを必ず含める（含めないと次回発動しない）
- 手順は「その業務を知らない人が読んでも実行できる」粒度で書く
- 対象タスクの KNOWLEDGE.md にあるハマりポイントは、スキルのセルフチェック節に反映する
