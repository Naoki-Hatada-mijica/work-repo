# サマリ出力フォーマット

以下をそのままコードブロック（テキスト）で出力する。Canvas や添付ファイルは使わない。

**重要（行間）:**
- セクション（【基本情報】【スキル・経験情報】【希望条件】）の間には空行を1行入れる
- ただし **同セクション内の項目（`■` や `・` の各行）は空行を空けずに連続** で並べる
- 冗長な空行を挿入しないこと
- **`■スキル・経験サマリー` の直後には空行を 1 行入れ、続けて `[スキルシート]` ラベル行を置き、URL は次行に改行して記載する**（ラベルと URL を同じ行に詰めない）

> **【希望条件】の4項目は FreelanceBase の希望条件フィールドと 1:1 対応させる**（`希望の作業場所`=work_location / `希望の作業内容`=work_content / `希望の作業時間`=work_time / `その他希望`=other_wish）。**ラベルは上記の表記どおり**にする（「その他**の**希望」ではなく「その他希望」）。稼働日数（週●日）や稼働形態は希望条件テキストではなく【基本情報】の該当行に書く。

## フォーマットのルール

### 必須
- [概要] は「・」で区切り、**最大7項目・1行40字以内**で記載する
  - 単なる列挙ではなく、経験や実績と関連づけて強みが伝わる表現にする
  - 例: 「・PythonとAWSを活用した大規模データ処理基盤の設計・構築経験」
- [経験スキル] はすべてのスキルを網羅し、カテゴリごとに整理する
  - 例: 「AWS → AWS（EC2/S3/Lambda/RDS）」のように詳細技術要素を明確化
  - **言語とフレームワークは「言語（FW1/FW2/...）」のスラッシュ区切り括弧表記**にする（カンマでなくスラッシュ）。例: `JavaScript（React/Next.js/Vue.js/Nuxt.js/Angular）`、`Node.js（Nest.js）/ PHP（Laravel/Zend Framework）`
  - **並び順は「直近であるほど」かつ「使用年数が長いほど」高スコアとし、高スコアの技術を各カテゴリ行の前半（左側）に置く**。これは言語・フレームワーク・ミドルウェア（DB / インフラ等）すべてに適用する。スキルシートの案件を時系列で見て、最新案件・長期利用の技術を左に寄せる
  - **候補者の主職種がシステムエンジニア／エンジニアの場合、コーダー・デザイナー寄りの周辺スキル（HTML / CSS / SASS / jQuery / Storybook / Photoshop / Illustrator 等）は出力しない**（ノイズになるため）。コーダー・デザイナー職の場合のみ記載する
  - **特定のツール名・製品名・ブランドを指さない汎用名称は載せない**（例: インフラ欄の `VPS` / `SSL` / `オンプレ` 等の総称。`Nginx` `Docker` `Terraform` `cron` のような具体ツール名は記載してよい）
- [人材担当コメント] は **100字以内** で人柄・強みを記載
- **提案単価**は「候補者の希望単価（税別）＋インボイス上乗せ」で算出して `￥●●●,●●●` を実額で埋める
  - インボイス**登録済**：希望＋**10万円/月** ／ インボイス**未登録**：希望＋**15万円/月**（いずれも税別）
  - インボイス登録有無は提案単価の内部参考のみ。**[概要]・[特記事項] などサマリ本文には書かない**
- **年齢は書かない**（生年月日から CRM が自動算出するため。テンプレートに ■年齢 行は無い）
- 日本国籍の場合は **【国籍】行ごと削除**
- 特筆事項がない場合は [特記事項] セクションを行ごと削除
- **希望条件に関わる事項（NG業種・NG案件・希望の方向性・優先順位・出社/リモート条件 等）は [特記事項] に書かず【希望条件】欄（特に ■その他希望）に集約する**。FreelanceBase 既存の希望欄と整合させる
- **【希望条件】の4ラベルは FreelanceBase のフィールドと 1:1 で一致させる**：`■希望の作業場所`(work_location) / `■希望の作業内容`(work_content) / `■希望の作業時間`(work_time＝コアタイム/時短等の時間帯希望、無ければ行削除) / `■その他希望`(other_wish)。**稼働日数（週●日）は希望条件テキストに入れず【基本情報】の ■稼働日数 に書く**（FBでも稼働日数は独立フィールド）
- **同セクション内に冗長な空行を入れない**（`■` や `・` の各行は詰めて並べる）

### 禁止
- 「フォーマットに沿って提案資料を作成しました」のような前置きを書かない
- Canvas や添付ファイルは使わない（チャット内の直接テキストのみ）
- **[経験職種]に「リーダー」「チームリーダー」を含めない**。CRM の occupation マスタには相当する独立カテゴリが無く、誤って「エンジニアリングマネージャー」（人事マネジメント職）にマップされる事故を防ぐため。
  - 本物の EM（評価・採用・1on1 等の人事マネジメント実務がある人）に限り「エンジニアリングマネージャー」と記載する
  - チームを引っ張った経験は [人材担当コメント] や [概要] で表現する

### 欠落時
- ソースから読み取れない項目は **「不明」** と明記（空欄にしない）
- 出力末尾に「使用したソース」のメタ情報を付記する（どのソースが欠落していたか分かるように）
