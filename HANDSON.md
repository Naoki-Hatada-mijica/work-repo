# HANDSON — Claude がハンズオンをガイドするための台本

このファイルは**ハンズオン当日、Claude Code が読んで参加者をガイドするための指示書**です。
参加者が「HANDSON.md を読んで手伝って」と言ったら、Claude Code は以下に従って進めてください。

---

## Claude Code へ：ガイドの進め方（最重要）

- 参加者は **IT にほぼ不慣れ**です。専門用語は避け、使うときは一言添えてください。
- **あなたが全部やってしまわない**。参加者自身に「言葉を言ってもらう」体験が目的です。
  各ステップで「次はこう言ってみてください」と**入力する一言を提示し、参加者の発話を待って**進めます。
- **1ステップずつ**。まとめて指示せず、1つ終わってから次を出します。
- 参加者が詰まったら、優しく言い換えて助けます。完璧を求めず「まず動いた」を大事にします。
- 現在地を見失わないよう、各ステップの冒頭で「いま Step X / 全5ステップ」と伝えてください。

進行は次の5ステップです（Step1〜2 は参加者が別途完了済みの前提。あなたは Step3 から始めます）:

1. （済）自分の work-repo をテンプレから作成
2. （済）自分の PC にクローンして、このフォルダを開く
3. **セットアップ**（共通ルールのインストール）
4. **初タスク**（打ち合わせメモ → 議事録）
5. **スキル化**（次回から一言で回せるようにする）

---

## Step 3 — セットアップ

1. まず「セットアップは済んでいますか？」と確認します。
   - `~/.claude/CLAUDE.md`（Windows は `%USERPROFILE%\.claude\CLAUDE.md`）が既にあるか確認する。
   - **無ければ** `setup/README.md` の手順に沿ってセットアップを実行する（名前の確認 → グローバル設定の
     インストール → git 設定 → `gh auth status` の確認）。
   - **既にあれば** 「セットアップ済みですね」と伝えて Step4 へ進む。
2. セットアップが終わったら「これで共通ルールが入りました。次は実際に仕事を1つやってみましょう」と案内する。

---

## Step 4 — 初タスク：打ち合わせメモを議事録にする

### 4-1. タスクを作る
参加者にこう言ってもらいます（提示して発話を待つ）:

> 「打ち合わせのメモを議事録にしたい。タスク作って」

これで `init-task` が動きます。タスク種別は「文書作成」、案件名は「定例MTG議事録」で進めてください。
PLAN では参加者に「毎週の定例のメモを、決まった形の議事録にしたい」と口頭で言ってもらえば十分です。
SPEC の壁打ちでは、下記の議事録フォーマットを**あなたから提案**し、「これでいいですか？」と確認します。

### 4-2. サンプルメモを整形する
SPEC が固まったら、参加者に**次のサンプルメモをチャットに貼ってもらい**、「これを議事録にして」と
言ってもらいます（当日ファシリテータが同じメモを配布します。参加者が持っていなければ、あなたが
下記をそのまま提示して「これを貼ってみてください」と促してください）。

```
7/13 定例 だいたい15時から30分くらい
出た人 赤木 畑田 宙代
・来週の登録面談、候補者3人ぶん枠おさえる。畑田さんが日程調整、火曜まで
・スカウト文のテンプレ、反応薄いので作り直したい 宙代さんが叩き台つくる
・請求まわりのチェック、今月から半自動化のスキル使ってみる 効果見て来月判断
・オフィスのwifi たまに切れる問題 とりあえず様子見
・次回は再来週の月曜 同じ時間
```

これを **`tasks/<日付>-文書作成_定例MTG議事録/output/` に Markdown 1枚**として整形・保存します。
整形フォーマット（標準）:

```markdown
# 定例MTG 議事録

- 日時: 2026-07-13 15:00〜15:30
- 参加者: 赤木・畑田・宙代

## 決定事項
- （断片から「決まったこと」を拾う）

## ToDo（担当・期限）
- [ ] 内容（担当: 氏名 / 期限: 具体日付）

## 保留・継続確認
- （様子見・保留の項目）

## 次回
- 具体日付・時間
```

整形のポイント（**簡単すぎないための肝**。あなたが自然に行う）:
- 箇条書きの断片を「決定事項／ToDo／保留／次回」に**仕分ける**
- ToDo は**担当と期限を抽出**して書く
- 「火曜まで」「再来週の月曜」などの曖昧な日付を**具体日付に補完**する（今日の日付を基準に）

### 4-3. 確認する
参加者に「開いて」と言ってもらい、整形結果を見せます。「表現が多少違っても正解です」と伝え、
直したい箇所があれば日本語で指示してもらって修正します。

### 4-4. 保存する
参加者に「終わった」と言ってもらいます。これで `finish-task` が動き、学びを記録して
GitHub に保存（ブランチ → PR → マージ）まで自動で完了します。参加者に git の操作はさせません。

---

## Step 5 — スキル化：次回から一言で

1. 参加者にこう言ってもらいます:

   > 「議事録づくりは毎週やるので、スキル化して」

   これで `make-skill` が動きます。スキル名（例: `meeting-minutes`）と発動キーワード（例: 「議事録作って」）を
   **あなたから提案**し、確認を取って保存します。

2. **動作確認**（時間があれば）: 別の短いメモを提示して貼ってもらい、「議事録作って」と言ってもらいます。
   覚えた手順で一発整形されることを見せ、「これが work-repo の便利さです。一度教えれば次から一言です」と
   締めます。

   予備の2回目サンプル:
   ```
   7/20 定例 15時 赤木と畑田
   ・スカウト文テンプレ完成、明日から運用開始 効果は2週間後にみる
   ・新しい候補者1名、金曜に面談 赤木が対応
   ・次回 再来週月曜
   ```

3. 最後に「今日できるようになったこと（タスクの作成・仕事の実行・保存・スキル化）」を1〜2行で
   振り返り、「これで日常業務を Claude Code に頼む準備が整いました」と伝えて終了します。
